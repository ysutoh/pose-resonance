# Role & Context
あなたはシニア・コンピュータビジョン・エンジニアおよびUI/UXデザイナーです。
現在開発中のフルート奏者向けリアルタイム姿勢解析PWA「Pose Resonance」（React + Vite + TensorFlow.js / MoveNet）に対して、科学的根拠に基づいたパーソナライズとデータトラッキングの3つの新機能を統合実装してください。

ターゲットユーザーは、全日本吹奏楽コンクールのソロ部門等に出場するプロレベルのフルートソリストです。長時間の練習における疲労度や、自分自身の「一番良い音が出る状態」との差分をシビアにモニタリングするため、絶対値ではなく「自分専用の基準値（ベースライン）」からの相対評価が必要です。

# New Requirements (追加実装仕様)

以下の3つの機能を現在の解析ループとUIに統合してください。チャート描画には `recharts` または `chart.js` (react-chartjs-2) などの軽量ライブラリを使用してください。

## 1. パーソナル・キャリブレーション（ベースライン設定）機能
- **要件:** 絶対的な水平（0度）ではなく、ユーザーが一番調子良く吹けている時のフォームを「基準値」として記録し、そこからのズレを評価する。
- **仕様:**
  - 画面に「Set Baseline（基準を設定）」ボタンを配置。
  - ボタン押下後、3秒間の「肩の傾き角度」と「頭の傾き角度」の平均値を算出し、State（可能ならlocalStorageにも）に保存する。
  - 警告UI（画面が赤くなる等のフィードバック）の条件を、「絶対値で±15度」から「保存したベースラインから±5度ズレた場合」に変更する。

## 2. 「頭部と脊椎のオフセット」測定機能（代償動作の検知）
- **要件:** 疲労時にフルートへ口を届かせようとして首が前に出たりねじれたりする「代償動作」を検知する。
- **仕様:**
  - 左右の肩の座標から「首の付け根（中心点）」を計算する。
  - 鼻（または左右の耳の中心）のX座標と、首の付け根のX座標のズレ（ピクセルまたは比率）を計算し、「Neck Offset（首のズレ）」として算出する。
  - このオフセット値が一定の閾値を超えた場合に、UI上で警告を表示する。

## 3. セッション時系列トラッキング機能（疲労の可視化）
- **要件:** 練習セッション中の姿勢の変化を記録し、グラフ化して疲労によるフォーム崩壊を可視化する。
- **仕様:**
  - 録画（または解析）開始から終了までの間、1秒ごとに「肩の角度のズレ」「頭の角度のズレ」「首のオフセット値」を配列として記録する。
  - 「End Session（セッション終了）」ボタンを追加。
  - セッション終了後、記録した時系列データを折れ線グラフ（Line Chart）として画面下部またはモーダルで表示する。X軸は時間、Y軸はベースラインからのズレ（度数/オフセット）とする。

# Execution Rules
- `npm install recharts` 等、必要なチャートライブラリをインストールすること。
- 既存の `requestAnimationFrame` による推論ループのパフォーマンスを落とさないよう、時系列データの記録（1秒ごと）はスロットリングや `setInterval` 等で適切に分離すること。
- Reactコンポーネントが肥大化する場合は、適宜ファイル分割（例：`Dashboard.jsx`, `SessionChart.jsx`）を行うこと。
- 全てのコードを出力し終えたら、動作確認のための手順を案内して完了とすること。
